# Title And Thumbnail

Use this file to package the video around one clear promise.
Title and thumbnail should be designed together.

## Title Rules

Prefer:
- simple verbs
- concrete danger
- one memorable twist
- readable nouns over technical terms

Strong families:
- `AI learns to <verb>`
- `I taught an AI to <verb>`
- `Can this AI survive / escape / carry / dodge <threat>?`
- `The AI kept failing until it discovered <twist>`

Avoid:
- acronyms in the title
- architecture names
- reward-shaping language
- titles that describe the build process more than the behavior

## Thumbnail Rules

Prefer:
- one protagonist
- one target or threat
- one strong contrast
- minimal or no text
- a frame that still reads on mobile

Common contrast types:
- tiny hero vs huge hazard
- almost-success vs obvious danger
- clumsy start vs elegant finish
- calm object vs glowing catastrophe

Avoid:
- multiple tiny agents with no focal subject
- dark murky frames
- screenshots where success and failure look the same
- text-heavy thumbnails that do the title's job

## Pairing Check

Before locking the package, ask:
- Does the title promise what the thumbnail visually shows?
- Would the thumbnail still make sense without the title?
- Would the title still be interesting without ML jargon?

If any answer is no, simplify the package.

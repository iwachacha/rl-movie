# RL Movie - Project Concept

## What This Project Is Trying To Make

RL Movie is a project about building reinforcement learning simulations that are worth watching as videos, not just worth evaluating as experiments.

The core goal is to show AI behavior that is readable, surprising, funny, eerie, or impressive even for viewers who do not care about RL terminology.

This project values the moment when an agent looks slightly unhinged, accidentally inventive, or strangely competent.
We want the audience to feel:

- "I instantly get the rule."
- "That failure was hilarious."
- "Why did the AI do that?"
- "Wait, it is actually getting good."

## Core Creative Direction

- Prioritize watchable behavior over benchmark novelty.
- Treat failure, instability, and weird strategies as part of the entertainment.
- Prefer AI that is memorable and hard to look away from over AI that is merely efficient.
- Make the learning process itself part of the appeal, especially when the agent behaves in a broken, chaotic, or unexpectedly clever way.
- Use LLMs, Codex, and AI agents as production accelerators and implementation partners, while keeping the on-screen focus on the simulated agent and its behavior.

## What "Good" Looks Like

A strong RL Movie scenario should:

- be understandable within a few seconds
- have one clear learning loop in the first version
- produce visible cause and effect without requiring debug UI
- create entertaining failure states, not just successful outcomes
- look richer than its implementation complexity by using smart staging, camera work, and free Asset Store content
- leave room for emergent behavior that feels a little uncanny or absurd

## Build Philosophy

- Take the shortest path to a strong-looking result.
- Use currently-free Unity Asset Store assets aggressively when they improve readability, spectacle, or atmosphere.
- Let AI agents help with implementation quality and speed.
- Keep V1 small, readable, and trainable before expanding scope.
- Choose scenarios that look advanced on video even if the underlying loop is simple.

## Content Bias

The project is especially interested in:

- funny or deranged learning failures
- agents discovering strange exploits or awkward strategies
- scenarios that look more intelligent than they are because the staging is strong
- compact simulations with clear goals, danger, and outcomes
- highly legible "AI learns a thing" videos with strong first-10-second hooks

## Things To Avoid

- technically clever scenarios that are visually flat
- rules that require long explanation before the viewer understands the scene
- reward shaping complexity that does not show up on screen
- overbuilding custom content when free assets can get there faster
- jumping into giant research-heavy systems before the simple version becomes watchable

## Long-Term Ambition

The near-term goal is to make short, clear, memorable RL videos with strong visual hooks.

The mid-term goal is to push into more ambitious multi-agent scenarios where conflict, cooperation, and weird social behavior start to emerge.

The long-term goal is to explore persistent worlds, open-ended training, and ecosystem-like simulations with no hard time limit, where viewers can watch agents develop habits, niches, and surprising patterns over time.

## Simple North Star

If a choice is unclear, bias toward this:

Make the AI more watchable, more readable, and a little more insane.

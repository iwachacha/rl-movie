# Scene Visual Guide

## 概要

AIエージェントとMCPを使ってUnity上の学習用シーンを構築・装飾する際、単に「アセットを配置して綺麗にして」と指示するだけでは、全体的に平坦でチープな（とりあえずオブジェクトを並べただけの）仕上がりになりがちです。

本ドキュメントでは、AIエージェントにダウンロード済みアセットを使わせつつ、高い視覚的クオリティ（Visual Quality）と動画としての視認性（Readability）を両立・担保させるための具体的な指示方法とガイドラインをまとめます。

このガイドは、`c:\rl-movie\.agents\skills\unity-scene-visual\SKILL.md` の内容と各種リファレンス（`style-lanes.md`, `visual-pass-order.md`, `lighting-recipes.md` 等）に基づいています。

## 質を担保するための5つのアプローチ

### 1. 専用の視覚ルール（スキル）を明示的に強制する

プロンプトの冒頭で、プロジェクトに定義されている「視覚的ルールと指針」をAIに読み込ませます。これにより、一般的な「Cinematic」のような曖昧な理解を防ぎ、プロジェクトの基準に沿わせることができます。

**指示の例:**
> 「作業に入る前に必ず `c:\rl-movie\.agents\skills\unity-scene-visual\SKILL.md` およびその関連リファレンスを読み、RL（強化学習）動画としての視認性と品質基準に従ってシーンを構築・調整してください。」

### 2. 「ビジュアル・レーン（方向性）」を1つに絞って指定する

曖昧なテーマ指定は、関係のないテイストのアセットやライティングの混在（スタイルの破綻）を招きます。構築を始める前に、必ず以下の3つの「レーン」から1つを明確に指定します。

*   **Broadcast Arena（デフォルト推奨）:**
    *   **特徴:** 床や壁がスッキリしており、コントラストが高い。eスポーツや競技のような明確さ。
    *   **適材:** 障害物コース、タイムアタック。エージェントと目標、危険地帯が最速で視聴者に伝わるスタイル。
*   **Stylized Setpiece:**
    *   **特徴:** アトモスフィア（雰囲気）重視。特定のカラーパレットやライティング、霧を活かしたリッチな絵作り。
    *   **適材:** 環境そのものが動画のフック（見どころ）になる場合。
*   **Simulation Documentary:**
    *   **特徴:** 落ち着いたエレガントなトーン。システム全体を淡々と観察させるような洗練されたデザイン。
    *   **適材:** 派手なアクションより、群れや創発的な賢さを観察するような動画。

### 3. アセット配置（Dressing）に「制限」を設ける

AIはアセットを渡されると、空間全体に均等にばらまく傾向（Prop Spam）があります。これを防ぐために空間の使い方を明確に指示します。

*   **プレイングレーン（動線）の確保:** エージェントの動きや意思決定が行われる中央エリアには余白（ネガティブスペース）を残す。
*   **クラスタリング（Clustering）:** アセットは均等に散らさず、部屋の隅や端、壁際に「固めて」配置させる。
*   **スケール感と接地感の担保:** オブジェクトが浮いて見えないよう、Contact Shadow（接地影）の設定や、大きさが一目でわかる基準となるオブジェクト（スケールキュー）を配置させる。

### 4. 調整の手順（Visual Pass Order）を細分化・段階化させる

MCPでのUnityの操作は、一度に大量の変更を行うと高い確率で意図せぬ結果を生みます。AIには「以下の順番ごとに区切って（必要なら都度画像や確認を挟んで）作業して」と指示します。

1.  **シルエットと色（Silhouette & Color）:**
    *   エージェント、目標、危険物、背景（床・壁）の色や明暗がしっかり分かれているか。同系色で背景に溶け込んでいないかを一番初めに確認させる。
2.  **接地とスケール（Plane & Contact）:**
    *   浮いているアセットを修正し、床と壁の境界を明確にする。
3.  **ライティングの階層化（Lighting Hierarchy）:**
    *   順光（Directional Light）一本槍ではなく、メインのキーライトと、暗部を適度に持ち上げるフィルライト（Fill light）を組み合わせさせる。必要なら主役を立たせるリムライト（Rim light）だけを追加。
4.  **アセットの装飾（Dressing & Background）:**
    *   ここで初めて、先述の端や角にアセットを配置する装飾工程に入らせる。
5.  **ポスト処理（Volume / Post Polish）:**
    *   暗闇や過剰なブルーム（Bloom）でごまかさせない。微妙なTone Mapping（ACES）やアンビエントオクルージョン（AO）など最小限の調整のみ適用させる。

### 5. 「ヒーローショット（カメラ視点）」を固定させる

「どこから見てもそれなり」を目指すと、最終的な動画画面としては平坦になります。「録画に使う予定のメインカメラ（例: `defaultCameraView` や `RecordingHelper` のカメラ）の画角」を基準とし、そこからの収まりが最も美しくなるように配置やライティングを調整させます。

## AI向け推奨プロンプト・テンプレート

AIエージェントにシーン作成とクオリティアップを依頼する際は、以下のテンプレートをコピー＆ペーストして活用してください。

```text
現在あるダウンロード済みアセットを使って、MCP経由でシーンのビジュアルクオリティを底上げしてください。
進行にあたっては `unity-scene-visual/SKILL.md` のルールを厳守してください。

1. ビジュアルレーンは [Broadcast Arena / Stylized Setpiece / Simulation Documentary] とします。
2. 録画用カメラからの見栄え（ヒーローショット）を最優先してください。
3. アセットを均等に散らかさず、プレイエリアは余白を残し、端や角にクラスタリングして配置してください。
4. 作業は以下のVisual Pass Orderの順で段階的に行い、ステップごとに報告とセーブを挟んでください。
   [色とシルエットの分離] → [接地感の修正とスケール配置] → [ライティング階層（Key/Fill）構築] → [小道具のクラスタリング配置] → [ポスト処理（Bloom等を抑えた洗練）]
```
